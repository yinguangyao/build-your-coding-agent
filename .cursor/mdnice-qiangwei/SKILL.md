# mdnice 蔷薇紫主题 · Markdown → WeChat HTML 转换器

## Overview

把 Markdown 文章一键转成带内联样式的 HTML，样式严格匹配 mdnice 的「蔷薇紫」主题（主色 `#8064a2`）。输出可以直接全选复制粘贴到微信公众号编辑器，无需再用 mdnice 在线编辑器。

**核心约束**：微信公众号编辑器只认 inline CSS，会过滤 `<style>` 标签和 class——所以所有样式必须写在元素的 `style=""` 属性里。

## Quick Reference

```bash
# 标准用法：输出到 .html 同名文件（同目录）
python3 .claude/skills/mdnice-qiangwei/convert.py path/to/article.md

# 自定义输出
python3 .claude/skills/mdnice-qiangwei/convert.py article.md -o output.html

# 自定义主色（默认 #8064a2 蔷薇紫；改成其他十六进制色即可换主题）
python3 .claude/skills/mdnice-qiangwei/convert.py article.md --color "#ff6b35"

# 关掉中文标点自动转换（保留 markdown 源里的 ASCII 标点）
python3 .claude/skills/mdnice-qiangwei/convert.py article.md --no-punct

# 输出到 stdout（便于管道）
python3 .claude/skills/mdnice-qiangwei/convert.py article.md --stdout

# 转完直接在浏览器打开
python3 .claude/skills/mdnice-qiangwei/convert.py article.md && open article.html
```

## 中文标点自动规范化（默认开启）

当半角 ASCII 标点出现在中文字符旁边时，自动转成全角中文标点。这是微信公众号文章的强约束 —— 中文段落混半角逗号读起来不舒服。

| 源 markdown | 转换后 |
|------|------|
| `凌晨,Vercel` | `凌晨，Vercel` |
| `推文:` | `推文：` |
| `飞起了吗?` | `飞起了吗？` |
| `沉默了!` | `沉默了！` |
| `不卖课;不讲道理` | `不卖课；不讲道理` |
| `(显式能力)` | `（显式能力）` |

**严格不转换的位置**：

- 代码块（` ``` ` 包围）—— 代码内 `def foo(a, b):` 保持原样
- 行内代码（`` ` `` 包围）—— `print("hello, world")` 保持原样
- 纯英文上下文 —— `(function)`、`hello, world` 不动
- markdown 语法字符 —— `[text](url)` 的 `(` `)` 不会破坏链接

如需完全禁用此行为，用 `--no-punct` 开关。

## 支持的 Markdown 语法

| 语法 | 渲染样式 |
|------|---------|
| `# 标题` | 居中 + 紫色短下划线（h1 主标题，22px） |
| `## 标题` | 左对齐 + 左侧 4px 紫色竖条（h2 章节，20px） |
| `### 标题` | 居中 + 紫色短下划线（h3 子标题，17px） |
| `**粗体**` | 紫色加粗 `#8064a2` |
| `` `行内代码` `` | 浅灰底 `#f4f4f5` + 黑字 + 等宽字体 |
| ` ```lang ` 代码块 ` ``` ` | 深色 Atom One Dark + 顶部 macOS 三圆点 |
| `> 引用` | 浅灰底 + 左侧 4px 紫色竖条 |
| `- 列表` / `* 列表` | 原生 ul + 实心圆点 |
| `1. 列表` | 原生 ol + 阿拉伯数字 |
| `---` | 居中短灰色分隔线 |
| `[文字](URL)` | 紫色下划线 |
| `![alt](URL)` | 居中图片（max-width: 100%） |

## 样式细节（可调参数）

打开 `convert.py` 顶部的 `STYLE` 常量区即可调整：

- `PRIMARY = "#8064a2"` —— 主色（蔷薇紫）
- `TEXT = "#3f3f3f"` —— 正文颜色
- `BG_GRAY = "#f7f7f7"` —— 引用块背景
- `BG_CODE_INLINE = "#f4f4f5"` —— 行内代码背景
- `CODE_BG = "#282c34"` —— 代码块背景（Atom One Dark）
- `FONT_BODY` / `FONT_MONO` —— 正文 / 等宽字体栈
- 字号、行高、margin 等都集中在脚本顶部

换主题色只需改一行 `PRIMARY`，全文所有强调元素会同步变色。

## 起公众号标题（AGI Hunt 风格）

**触发**：用户说「帮我起个标题」「想几个爆款标题」「这篇文章叫什么好」时直接走本节。

### 标题硬规则

- **字数**：12-30 字，多分句
- **必有**：至少 1 个数字 **或** 3 个并列短词 **或** 1 个感叹号
- **可有**：「」直角引号包黑话词、产品/公司名大写英文
- **绝无**：宫斗式动词（踹下/血洗/深夜放炸弹 → 那是新智元风格，互斥）、Z 世代抖音梗（yyds/破防/炸裂）、长串 emoji

### 8 套路速查

| # | 套路 | 模板 | 适用场景 |
|---|---|---|---|
| 1 | 刚刚战报 | `刚刚，X [动作]，[3 个并列短词]` | 数小时内新闻 |
| 2 | 突发反差 | `突发！X [出乎意料的事]` | 反预期事件 |
| 3 | 重磅数字 | `重磅！[N] 页 [产品名]曝光！[情绪关键词]！` | 深度爆料 |
| 4 | 终于攻下 | `[主体]终于攻下了 X！` | 长期 anticipation |
| 5 | 数字碾压 | `X 刷新 Y：N% 碾压所有 Z，[反问]？` | benchmark 突破 |
| 6 | 我让实测 | `我让 X 当裁判，横评 Y 和 Z` | 第一人称实测 |
| 7 | 疯了/太野 | `疯了……X！[副标题]` | 离谱事件 |
| 8 | 这才是 | `X，这才是探索 Y 的正确姿势！` | 路线之争 |

### 标准输出格式

每篇文章给 **5 个候选**，每个候选必带四项标注：

```
### 🎯 候选 N（套路名）

> **「<标题文本>」**

- **套路**：套路 # + 模板名（如：模板 7 疯了/太野）
- **字数**：N 字 ✓/✗
- **事实硬度** ★★★★★（有几个可核实事实/数字）
- **戏剧硬度** ★★★★★（情绪锚词强度）
- **风险**：⚠️ 例如缺主体名、克制路线戏剧度低、词义冲突 / 无
```

最后给一个推荐顺序（一般推荐 1/3，技术深度号推荐 2/5），让用户选。

### 黑话词库（直接调用）

```
副词：刚刚 / 终于 / 重磅 / 突发 / 疯了…… / 太野了
动作：碾压 / 攻下 / 刷新 / 直接拿下 / 强行合并 / 投下炸弹
术语：SOTA / Agent / 长上下文 / MoE / 推理 / 开源
情绪：神器 / 神器级 / 王牌 / 临界点 / 起飞 / 自举
```

## 生成封面图（dreamina text2image）

**触发**：用户说「生成封面图」「画个公众号封面」「来张头图」时直接走本节。

### 调用前自检

```bash
# 1. 确认登录
dreamina user_credit
# 2. credit 不足（< 20）先充值或换号
```

### 一行命令

```bash
dreamina text2image \
  --prompt="<视觉 prompt>" \
  --ratio=16:9 \
  --resolution_type=2k \
  --model_version=5.0 \
  --poll=120
```

**参数选择**：

| 参数 | 推荐值 | 备注 |
|---|---|---|
| `ratio` | `16:9` | 公众号封面横版标配；方形头图用 `1:1` |
| `resolution_type` | `2k` | 公众号阅读够用；不要选 `4k`（浪费 credit） |
| `model_version` | `5.0` | 当前最强；备用 `4.6` |
| `--poll` | `120` | 给足 2 分钟生图时间，太短会 fallback 到手动 query |

### prompt 三段式模板

```
[主体视觉隐喻] + [配色：深紫色 #8064a2 + 近黑背景] + [风格锚词：电影感、4k 高细节、不出现文字]
```

**写 prompt 三条铁律**：

1. **不要在图里要文字**——任何「文字 / 标题 / 字 / 标语 / logo 字」描述会出错字。结尾必须加 **「画面中不出现文字」**。
2. **用具象隐喻替代抽象概念**——「AI 取代人类」不行，「代码瀑布 + 人类剪影按 Merge 按钮」可以。
3. **配色锁死蔷薇紫 + 暗背景**——和 HTML 主题一致，整篇视觉统一。

### 三个范例（按文章类型选）

**1. AI 速报体（科技工业感）—— 适配 agi-hunt-style 写的文章**

```
暗调电影感概念插画：[隐喻主体]，[关键数字符号]，[人物/工具元素]，
整体配色深紫色 #8064a2 高光配近黑背景，
工业冷峻略带孤独感，4k 高细节，画面中不出现文字
```

实例（本篇 Bun 文章）：
```
暗调电影感概念插画：一道发光的代码瀑布从夜空云端倾泻而下，
瀑布中漂浮着巨大的数字 6755 和 Rust 螃蟹符号；
瀑布脚下一个孤独的人类剪影正按下一个紫色发光的 Merge 按钮，
整体配色以深紫色 #8064a2 高光配近黑背景，
工业冷峻又略带孤独感，电影级打光，4k 高细节，画面中不出现文字
```

**2. 技术深度体（克制极简）—— 适配偏严肃的技术分析文**

```
极简几何插画：[抽象图形主体]，深紫色与黑色块状构图，
柔和体积光，包豪斯风格 / 等距视角，画面中不出现文字
```

**3. 行业评论体（人物对峙 / 路线之争）—— 适配争议性话题**

```
双人剪影对峙概念插画：[人物 A] vs [人物 B]，
背景是 [事件场景]，紫色雾气环绕，电影感打光，
4k 高细节，画面中不出现文字
```

### 后续步骤（生图完成后）

dreamina 输出 JSON 含 `image_url`（签名链接，过期前必须下载）：

```bash
# 从 output 提取 image_url，下载到 wechat-docs 目录
curl -L -o /path/to/article-cover.png "<image_url>"

# 浏览器打开预览
open /path/to/article-cover.png
```

⚠️ **签名 URL 会过期**（通常 30 天，但保险起见生成后立即下载）。
⚠️ **不要 image2image 改图**——不满意直接重写 prompt 再 text2image，迭代更快。

### 避坑清单

| 问题 | 原因 | 解决 |
|---|---|---|
| 出图带错字 / 乱码 | prompt 里含「文字 / 标题 / logo」描述 | 全删，加「画面中不出现文字」 |
| 出图颜色不对 | 没锁主色 | 显式写 `#8064a2` 或「深紫色」 |
| 出图太亮 / 风格不统一 | 风格锚词太散 | 主体之外只留 2-3 个风格关键词（电影感 / 4k / 暗调） |
| 等了很久没结果 | `--poll` 太短 | 用 `dreamina query_result --submit_id=<id>` 手动查 |
| credit 突然扣很多 | 选了 `4k` | 公众号封面 `2k` 已绰绰有余 |

## Common Mistakes

| 问题 | 原因 | 解决 |
|------|------|------|
| 微信里粘贴后样式丢失 | 用了 class / `<style>` 标签 | 本脚本输出全 inline 样式，复制时确保全选所有元素 |
| 代码块超出屏幕 | 微信容器宽度限制 | 脚本已加 `overflow-x: auto`，长代码可横向滚动 |
| 中英文标点混杂 | markdown 源文件半角符号 | 写作时直接用全角 `，：？！`，脚本不会自动转换 |
| 表格不渲染 | 微信公众号编辑器对 `<table>` 支持有限 | 表格转成 HTML 后微信通常会正常渲染，但复杂表格请避免 |

## 实现要点

脚本是单文件、零外部依赖（不依赖 `markdown` / `mistune` 等库），用正则 + 行级状态机解析 Markdown。

支持的转义和边界情况：
- 代码块内的 HTML 字符自动转义（`<` `>` `&`）
- 行内代码内不再处理 markdown（避免 `` `**a**` `` 把里面的 `**` 当成粗体）
- 段落允许跨多行（自动合并）
- 列表项允许嵌套换行
- HR 必须独占一行（`---` 前后空行）

不支持的（避免复杂度）：
- 嵌套列表
- 表格的多列对齐语法 `:---:`
- HTML 块直通（如果 md 里写 raw HTML 会被原样输出，不再处理）
- 任务列表 `- [x]`

如果需要这些特性，直接改 `convert.py` 加 case。
